    播客制作手册

    播客的形态和特点¶

    • 播客访谈分为2种：故事型和逻辑分析型
    • 前者和个人经历，发生的事情等等有关；后者和事件分析、行业分析有关
    • 播客是线性的，无论哪种类型，听众都会被天然的问题所推动
    • 最重要的没有之一：找到 Driving Question 核心问题
    • 并没有完美的公式，只要外表精致，内在和谐就可以
    • 采访和聊天是完全不一样的思路，访谈最大化采访对象的认知，聊天最大化成员之间的碰撞
    • 利用峰值体验：开头要抓人，中间有记忆点，结尾最好能带来启发
    • 面向自己创作
    • 基于自己的理解去表达，而不是为了某个观点而表达，否则就会生硬不自然
    • 社群的基因在于群体智慧的共同创作，要发挥这一点优势
    • 面向听众制作
    • 听众喜欢具体的，比如有深度的个体经验（bottom-up）、有具体案例的宏观分析（top-down）
    • 听众需要context，站在听众的角度介绍嘉宾、前因后果、专业词汇
    • 好的访谈应该是什么样的
    • 是一场对话，而不是僵硬的QA
    • 准备充分，但也临场随机应变
    • 有重点有结构
    • 能有让人记住的点

    选题¶

    • 为什么听众要关心这个话题
    • 嘉宾，以及嘉宾的表达能力
    • 是否别的播客已经做了，其角度和结构是什么
    • 明确角度，聚焦单点
    • 不要停留在现象，要可操作

    前采¶

    • 聊天类
    • Host 先列出大体框架，一起头脑风暴完善结构+细化要点
    • Host 要做好功课，阅读尽可能多的资料，更好地控场
    • 参考 灰度上线08-互联网和春节，如何为彼此创造价值？
    • 访谈类
    • 之前是否听播客
    • 关注这个领域多久了，有哪些方面的研究
    • 最想表达的信息和观点是什么
    • 充分沟通预期（大致的形式、内容、时长、工具）
    • 参考 灰度上线07-短视频

    提纲¶

    • 可能的结构
    • 讲故事体：个人观察/经历 → 对事物的描述/分析/评论 → 总结规律或趋势 → 结尾回到小故事点题
    • 洋葱模型：成就/title → 故事 → 经验/规律 → 价值观
    • checklist
    • 这一期播客最关键的那个问题是什么？
    • 你问的问题彼此之间有逻辑关系吗？能够组成像脑图或目录一样的结构吗？
    • 放在开头的问题是否能吸引住听众的注意？
    • 你要挖掘的故事有哪些？彼此之间什么联系
    • 假设你的听众很无知，有没有一些基本概念是需要科普的？
    • 想象一下你是在为听众提问
    • 设计好的问题可以写逐字稿，干净利落的问出来

    录制¶

    • 远程
    • 3 个设备：一个录音设备（手机或者录音笔），一个通话设备（Pad/电脑/手机），一条耳机
    • 录制环境：找一个没有回声，无噪音，无打扰的房间录制
    • 固定录音设备在下巴附近的位置，录制过程不要触碰，不要暂停⏸，结束后再停止录制⏹
    • 10s 测试：录制前发一个 30s~1min 的试录文件给我们听一下效果
    • 发给嘉宾的文字说明 【经验】远程录音说明 🎤 ，视频说明 远程录制指南.mp4
    • 现场
    • 设备：手机（至少2个手机录音，放在中间）或 Zoom H1N（谁说话指向谁，注意距离不要喷麦）
    • 现场所有人录制为一个轨道

    剪辑¶

    • 软件
    • Audition，软件下载（mac，win），Mac安装教程，Windows安装教程
    • 苹果自带的 GarageBand
    • 步骤 步骤 说明 截图 合并 录制结束后汇总参与者的音频文件 导入 Audition 创建多轨文件，路径为【文件 -> 新建 -> 多轨会话 -> 修改命名 -> 确定】 可以使用AU自带的「播客模板」 对齐后导出为mp3，路径为【文件 -> 导出 -> 多轨混音 -> 整个会话】

    粗剪¶

    根据上下文结构，大段删掉不重要or重复的部分

    精剪¶

    • 删去口癖、口误、中断、明显的环境噪音等
    • 响度：所有轨道音量对齐到-9~-12db之间（底部音量条不要出现大面积红色），可对整条轨道or单个片段调节音量，确保每条轨道的音量一致
    • 降噪：为轨道添加降噪（如果效果比较奇怪就不加了）

    • 快捷键：

      • 常用：Cut=cmd+K，M=插入标记
      • 自定义：快速选中右侧全部片段，这样能够提高剪辑后拼接的效率，路径为【剪辑-键盘快捷键-搜“选中直到会话末尾的剪辑”-设置快捷键（我的习惯是shift+R）】

    开头结尾¶

    • 已有素材：声音logo、outro（可根据文字稿自行录制）
    • chrome插件下载音乐：网易云音乐、youtube video downloader

    导出¶

    • 导出为 mp3 44100HZ 16bit 的立体声，192Kbps CPR，路径为【文件 -> 导出 -> 多轨混音 -> 整个会话】
    • 把mp3上传到妙记，链接发到播客小分队里一起勘误
